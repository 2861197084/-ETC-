# ETC 高速公路大数据平台 - 需求范围（与实现一致）

> 本文档仅保留当前仓库已实现/对齐的需求点；不落地或与实现不一致的内容已移除。

---

## 1. 核心场景

1. **实时监控指挥舱**：展示卡口分布、实时流量、压力预警、套牌/违章等事件，并使用**虚拟时间**模拟业务运行。
2. **数据查询中心**：支持按车牌/卡口/时间范围查询通行记录，并支持热数据 + 历史数据的渐进式加载。

---

## 2. 数据与时间模拟要求

- 虚拟时间基准：`2024-01-01 00:00:00`
- 时间倍率：`1s(真实) = 5min(虚拟)`
- 数据分层：
  - 历史数据（2023-12）：CSV → HBase ✅
  - 实时数据（2024-01）：CSV → Kafka → Flink → MySQL（热数据）✅，并归档到 HBase ✅

---

## 3. 查询路由要求

- 近 7 天热数据：MySQL ✅
- 历史查询：HBase ✅
- 统计汇总：Redis ✅

---

## 4. 数据保留策略

- MySQL 热数据默认保留 7 天（按虚拟时间），后端定时清理 ✅
